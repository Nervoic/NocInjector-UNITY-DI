## Описание
Контексты созданы для разделения областей видимости контейнеров. К каждому контексту привязан контейнер. Зависимости в данных контейнерах ничего не знают о зависимостях в контейнерах других контекстов

## Виды контекстов
Каждый контекст представлен под общим интерфейсом **IContext**
- **MonoContext** - контекст, привязанный к жизненному циклу Unity. Создается при наследовании **MonoContext**.

Пример **MonoContext** -
```csharp

public class CustomContext : MonoContext 
{
    protected override void OnInitialize() 
    {
        // Контекст не будет уничтожен при смене сцены
        DontDestroyOnLoad(gameObject)
    }
}
```

- **Context** - стандартный контекст, жизненный цикл которого полностью контролирует пользователь.

Пример **Context** -

```csharp
public class ContextStorage : MonoBehaviour
{
    public void Start() 
    {
        var context = new Context()
    }
}
```
Внутреннюю структуру контекстов можно посмотреть, заглянув в XML-документацию внутри кода.

## Инициализация контекстов
Для установки зависимостей используются специальные классы - **Installer-s**.

Как и в случае с контекстами, существуют 2 вида **Installer-s**, они представлены под общим интерфейсом **IInstaller**

- MonoInstaller - **Installer**, представленный в виде компонента. Нужен для инсталляции зависимостей в **MonoContext**, но также может быть передан при создании **Context**


- **Installer** - Стандартный **Installer**, не может быть использован для инсталяции зависимостей внутри **MonoContext**, но может быть использован при создании **Context**
