## Описание
Иньекция зависимостей используется для автоматического получения зависимостей теми классами и компонентами, которые этого требуют

## Как проходит иньекция
Вообщем иньекция происходит через методы, или конструкторы. NocInjector не поддерживает иньекцию в поля и свойства. Для того чтобы в метод были внедрены зависимости, нужно использовать атрибут **[Inject]**

Пример использования атрибута **[Inject]** - 

```csharp

public class PlayerController : MonoBehaviour
{
    private PlayerSpeedInfo _speedInfo;
    
    [Inject] 
    private void Construct(PlayerSpeedInfo speedInfo) 
    {
        _speedInfo = speedInfo
    }
}
```

Иньекция в конструктор происходит так же, только по атрибуту **[Inject]**

```csharp
public class PlayerController : MonoBehaviour
{
    private readonly PlayerSpeedInfo _speedInfo;
    
    [Inject] 
    public PlayerController(PlayerSpeedInfo speedInfo) 
    {
        _speedInfo = speedInfo
    }
}
```

## Конкретика иньекции
Иньекция происходит только из того же контейнера, в котором была зарегистрирована зависимость.
Если же в этом контейнере нет зависимости, которая должна быть внедрена, контейнер ищет эту зависимость в родительском контейнере(если при создании контекста данного контейнера ему был установлен родительский контекст).

## Иньекция в компоненты - 
Происходит только, если он зарегистрирован в каком либо контексте, и внедрение, как упомяналось ранее, происходит из контейнера того же контекста, где он был зарегистрирован, или его родительского контейнера.


Если компонент зарегистрирован как Singleton - иньекция происходит только в этот компонент

Если же компонент зарегистрирован как Transient, то при запросе данного компонента будет создан новый экземпляр GameObject данного компонента, и в этот только что созданный GameObject будут внедрены зависимости во все его компоненты.

## Стандартная иньекция -

Если зарегистрировать зависимость в контейнер, и она не является компонентом, то вы можете использовать атрибут **[Inject]** в конструкторе данной зависимости. Иньекция в методы в данном случае не будет работать.
